# near-eth-withdraw

Оффлайн-подписант для вывода **нативных NEAR** с **eth-implicit аккаунта** (`0x…`) —
того самого «внутреннего адреса», которым ты управляешь своим **EVM-приватником**
(near.com / Ethereum-кошельки на NEAR, стандарт [NEP-518](https://github.com/near/NEPs/issues/518)).

Скрипт подписывает перевод **локально** (ключ никуда не уходит) и выдаёт два аргумента
для метода `rlp_execute` контракта-кошелька. Отправку (оплату газа) делает любой обычный
NEAR-аккаунт через NearBlocks или near-cli.

> Кодировка сверена строка-в-строку с исходником контракта
> [`near/near-wallet-contract`](https://github.com/near/near-wallet-contract)
> (`wallet-contract/src`: `internal.rs`, `types.rs`, `lib.rs`).

---

## Когда это нужно

У тебя есть адрес вида `0x71C7…976F` (NEAR-аккаунт, а не Ethereum), на нём лежат
**нативные NEAR**, и обычного NEAR-ключа к нему нет — но есть **EVM-приватник**
(импортируется в MetaMask/Rabby). Это и есть eth-implicit аккаунт: им управляет
secp256k1-ключ через задеплоенный wallet-contract. Этот скрипт даёт вывести с него NEAR
самостоятельно, не дожидаясь саппорта.

---

## Как это работает (коротко)

1. Локально подписывается legacy-Ethereum-транзакция (`chainId 397` mainnet),
   в которую закодирован NEAR-перевод: `transfer(string receiver_id, uint32 yocto_near)`,
   `value = сумма_yocto / 1e6`, `to = keccak256(receiver_id)[-20:]`.
2. Подпись (RLP) кодируется в base64 → это `tx_bytes_b64`.
3. Любой аккаунт вызывает `rlp_execute(target, tx_bytes_b64)` на твоём `0x…` аккаунте
   и платит ~0.001 NEAR газа. Контракт проверяет подпись и делает перевод.

Защита: подпись может сделать только владелец ключа. Если что-то в blob не сходится
с `target` — контракт просто отклонит вызов (теряется только газ, деньги целы).

## Использование

### Шаг 1. Узнать nonce

Текущий nonce твоего аккаунта = метод `get_nonce` контракта (на NearBlocks → Contract →
`get_nonce` → Query). Первая в жизни транзакция = `0`, дальше растёт на 1.
Либо передай `auto` — скрипт сам прочитает его по RPC.

### Шаг 2. Подписать (локально)

PowerShell (Windows):
```powershell
$env:PRIVATE_KEY = "0xТВОЙ_EVM_КЛЮЧ"
node sign.js <receiverAccountId> <amountNEAR> <nonce>
$env:PRIVATE_KEY = ""
```

bash / macOS / Linux:
```bash
PRIVATE_KEY=0x... node sign.js <receiverAccountId> <amountNEAR> <nonce>
```

Пример (1 NEAR на implicit-аккаунт, nonce 0):
```bash
PRIVATE_KEY=0x... node sign.js alice.near 1 0
# nonce можно не считать вручную:
PRIVATE_KEY=0x... node sign.js alice.near 12.5 auto
# testnet:
PRIVATE_KEY=0x... node sign.js alice.testnet 1 0 --testnet
```

Скрипт печатает блок `VERIFY` (сверь `from` = твой `0x…` аккаунт) и строку
`{"target":"…","tx_bytes_b64":"…"}`.

### Шаг 3. (по желанию) Проверить blob

```bash
node verify.js "<tx_bytes_b64>" <receiverAccountId>
```
Расшифрует подписанную транзакцию и покажет: куда, сколько, какой nonce, отправитель,
и совпадает ли всё с `target`. Ключ не нужен.

### Шаг 4. Отправить `rlp_execute`

**Вариант А — NearBlocks (проще всего):**
1. Открой страницу **своего** аккаунта `0x…` → вкладка **Contract → Write**.
2. Сверху **Connect to Wallet** — подключи любой обычный аккаунт для оплаты газа
   (**не** сам `0x…`, у него нет обычного ключа).
3. Метод **`rlp_execute`**, кнопкой **Add** добавь два аргумента (оба Type = **String**):
   - `target` = receiverAccountId (точно как в скрипте)
   - `tx_bytes_b64` = base64 из вывода
4. Attached deposit = `0`, Gas = `100000000000000` (100 Tgas) → **Write** → подтверди.

**Вариант Б — near-cli-rs:**
```bash
near contract call-function as-transaction <ТВОЙ_0x_АККАУНТ> rlp_execute \
  json-args '{"target":"<receiver>","tx_bytes_b64":"<base64>"}' \
  prepaid-gas '100 Tgas' attached-deposit '0 NEAR' \
  sign-as <ГАЗ_АККАУНТ> network-config mainnet sign-with-keychain send
```

**Проверка успеха:** `get_nonce` увеличился на 1, средства пришли получателю.

## ⚠️ Важно

- **Не выводи «в ноль».** Аккаунт должен оставить себе немного NEAR на собственное
  хранилище, иначе перевод отклонится (`LackBalanceForState`). Оставь буфер ~0.05–0.1 NEAR.
- **`target` обязан совпадать** с `receiverAccountId`, под который подписан blob.
  Не совпадёт — будет ошибка (деньги целы, тратится только газ).
- **Перевод необратим.** Дважды проверь адрес получателя — что он твой/верный.

## FAQ

**Это безопасно? Скрипт не украдёт ключ?**
Ключ читается только из переменной окружения `PRIVATE_KEY`, нигде не печатается и не
отправляется в сеть. Подпись делается локально. Код открыт — прочитай `sign.js` (≈150 строк).
Можешь запускать с выключенным интернетом (тогда nonce укажи числом, не `auto`).

**Что такое `tx_bytes_b64`? Его можно кому-то показать?**
Это уже подписанная транзакция в base64. Приватного ключа в ней нет — только подпись.
Показывать безопасно. Любой может её проверить через `verify.js`.

**Откуда брать nonce?**
`get_nonce` на твоём аккаунте (NearBlocks → Query) или передай `auto`.

**Почему `value` в blob выглядит как 18 знаков, а не 24?**
В eth-транзакции 18 десятичных, а в NEAR — 24. Контракт умножает `value` на `1e6` и
прибавляет `yocto_near` (остаток). Поэтому `value = сумма_yocto / 1e6`. Скрипт считает сам.

**Можно ли отправить на биржу / именованный аккаунт (`name.near`)?**
Да, `receiverAccountId` может быть любым валидным NEAR-аккаунтом. Для бирж учитывай,
что им часто нужен MEMO/tag — на нативный перевод его не прицепить, уточняй у биржи.

**Кто платит газ?**
Тот аккаунт, которым ты подписываешь сам вызов `rlp_execute` (~0.001 NEAR). К балансу
выводимого `0x…` аккаунта это отношения не имеет.

**Перевёл на адрес, которым не владею. Вернуть можно?**
Нет. Переводы необратимы. Поэтому шаг 3 (`verify.js`) и сверка адреса — обязательны.
